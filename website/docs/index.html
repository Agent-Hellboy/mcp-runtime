Welcome

Deploy, govern, and observe MCP services on Kubernetes.

MCP Runtime gives platform teams a higher-level control plane for MCP delivery, access, policy, audit, and observability across internal MCP services.

Start with runtime docs Open API reference

Why MCP Runtime?

  • mcp-runtime owns setup, deployment, grants, and sessions as the control plane.
  • mcp-sentinel owns policy enforcement, audit, query, and observability on the request path.
  • Registry, ingress, and rollout workflows are part of the same runtime surface.
  • The split is clear enough to explain internally and practical enough to operate.

What this is not

Not an Istio-style mesh control plane or a generic proxy fleet manager

MCP Runtime sits above lower-level ingress, gateway, and service-mesh infrastructure. It standardizes MCP deployment, access, policy, and audit for MCP services rather than trying to become the network control plane for every workload in the cluster.

Core benefits

Benefit Description
Clear plane split Use runtime for lifecycle and cluster state, and sentinel for live MCP request governance, instead of mixing both concerns into ad hoc services.
Unified delivery path Use one CLI and one API surface for cluster setup, image flow, manifest generation, and MCP server rollout.
Real access model Separate grants and sessions keep trust ceilings, consent, expiry, and revocation out of deployment-only YAML.
Request-path enforcement Evaluate policy at tool-call time with identity headers, trust rules, and allow or deny audit decisions.
Bundled operator visibility Route audit and telemetry into sentinel ingest, processor, API, UI, and observability surfaces without adding a second product path.

Quick start

These commands are the current short path from repo checkout to a running platform flow.

make deps && make build-runtime

./bin/mcp-runtime setup
./bin/mcp-runtime status

# Optional:
# ./bin/mcp-runtime setup --with-tls
# ./bin/mcp-runtime setup --without-sentinel

./bin/mcp-runtime registry push --image my-server:latest
./bin/mcp-runtime pipeline generate --dir .mcp --output manifests/
./bin/mcp-runtime pipeline deploy --dir manifests/

Current scope

Alpha, but broad enough to evaluate

Deployment, routing, grants, sessions, gateway policy, and audit flow are implemented in the current repo. Sentinel supports bearer-token validation, but a full OAuth 2.1 authorization server and Dynamic Client Registration flow are still out of scope.
